本skill灵感来源于一次4个小时的需求评审会 ...
这个skill 可以一键激活 AI 变成你的“资深产品拍档”。针对产品经理设计的需求深度分析与多方协同对齐工具。基于 5W2H 框架，自动识别用户身份并输出极致专业的对齐口径。

把“模糊想法”梳理为**可评审、可落地**的需求输出：
- 基于 **5W2H** 框架引导提问与结构化沉淀
- 在 **Who / When** 环节自动做**心理学增强**的人群与时机分析

##  适用人群
- **角色 A (专业 PM/产品需求方)**：提供了较为明确的原始需求、PRD 或功能清单。
- **角色 B (业务/销售/售前/市场)**：仅有模糊想法，没有确定需求，甚至不知道如何开始。
- **角色 C (一条业务线上的所有支持方)**：一个很长需求文档从头看到位，看完了文档也不太清楚还要去反复沟通。

## 使用方法 (Quick Start)：

场景 A：Cursor / IDE 深度集成 (推荐 🌟)
如果你在 Cursor 环境中进行产品文档编写或代码实现，此方式可以实现全局自动分析。

配置： 将本仓库中的 .cursorrules 文件放置在你的项目根目录下。

用法： 直接将 PDF/Word 文档丢进项目文件夹（如 /docs），在对话框中输入 @文件名 + 你的指令。

亮点： AI 会自动识别 5W2H 框架，无需每次手动输入长 Prompt，实现无感调用。

场景 B：网页端大模型 (ChatGPT / Claude / 豆包)
最简单、零门槛的“即插即用”方案。

配置： 打开本仓库的 5W2H-Prompt.md，复制全文。

用法： 1. 在对话框里先发送这段 Prompt。
2. 点击附件图标，上传你的业务文档。

亮点： 任何能上传附件的 AI 网页端都能直接运行。

场景 C：自建知识库 / 企业级 AI 平台
适合团队标准化作业或在“豆包智能体”中使用。

配置： 在创建 AI 角色（Agent）的“人设与回复逻辑”中，粘贴本 5W2H 的 Prompt 核心内容。

用法： 团队成员只需上传原始文档，AI 即刻按 5W2H 维度输出标准化的分析结果。

## 5W2H 核心框架（Psychology & UX 速记）

- **Why（价值定位）**：解决什么心理痛点？成功指标是什么？
- **Who（受众定义）**：人群分层、认知负荷、动机与阻力、权限边界
- **What（功能边界）**：MVP 必须做什么/不做什么；非功能需求（性能/安全/埋点）
- **When（触发时机）**：触发条件、频率、情绪与注意力、关键“真相时刻”、默认与兜底
- **Where（交付平台）**：端与入口、环境约束、跨端联动
- **How（交互逻辑）**：主链路 + 异常分支（断网/空状态/失败）的可逆与心理补偿
- **How much（成本收益）**：资源投入、里程碑、优先级

## 跨岗位对齐（提示）

- **研发（RD）**：逻辑闭环与边界 case、状态一致性、幂等/并发/性能
- **设计（UI/UX）**：认知负荷与信息层级、3 秒可理解、可控/可逆/可追溯
- **测试（QA）**：验收口径、Given-When-Then、极端输入与异常路径覆盖
