# 🚀 5W2H-Skill-for-PM：高质感需求定义与对齐框架
> **基于心理学洞察与 AI 逻辑的产品经理需求工具包**

## 📖 项目简介
本项目旨在将传统的 5W2H 分析法工程化，为产品经理提供一套标准的“需求自检与对齐”Skill。通过结合 **心理学（用户行为动机）** 与 **UX 设计原则**，确保需求在提交给研发和设计时，具备极高的清晰度和可执行性。

---

## 🛠 5W2H 核心框架 (Psychology & UX Edition)

### 1. Why (价值定位)
* **核心自问：** 这个功能解决了什么心理痛点？（如：缓解焦虑、增加确定性）
* **衡量指标：** 上线后哪个数据会发生变化？

### 2. Who (受众定义)
* **用户画像：** 用户的认知负荷如何？是高敏感人群 (HSP) 还是大众用户？
* **角色权限：** 不同等级用户的操作边界在哪里？

### 3. What (功能边界)
* **核心路径：** MVP 版本必须包含什么？
* **非功能需求：** 性能、安全性、埋点要求。

### 4. When (触发时机)
* **环境因素：** 用户在什么情绪状态或物理环境下使用？
* **频率评估：** 是一次性设置还是高频高交互？

### 5. Where (交付平台)
* **环境约束：** 移动端/Web/AI 硬件？是否涉及跨端联动？

### 6. How (交互逻辑)
* **业务流图：** 异常分支（断网、空状态）的心理补偿设计。
* **AI 介入：** 如果是 AI 功能，如何处理生成式内容的不可预测性？

### 7. How Much (成本收益)
* **资源投入：** 预估开发量与业务产出的性价比。

## 🤝 跨岗位对齐 Skill (Alignment Guide)

| 对接对象 | 侧重点 | 必问的一个心理学问题 |
| :--- | :--- | :--- |
| **研发 (RD)** | 逻辑闭环、边界 Case | “如果用户在解析中途突然断网，系统状态如何保持一致？” |
| **设计 (UI/UX)** | 情感体验、视觉等级 | “这个页面的核心视觉锚点能否在 3 秒内缓解用户的选择焦虑？” |
| **测试 (QA)** | 验收标准、异常路径 | “当输入极其罕见的极端数据时，预期的系统心理反馈是什么？” |
